4. Accessing Remote Console

4.1 Dell iDRAC Configuration

Accessing iDRAC Web Interface

Step 1: Open Web Browser

URL: https://<iDRAC-IP>

Example: https://192.168.1.100

Step 2: Login Credentials

Default Credentials (Factory Reset):

  • Username: root
  • Password: calvin (or label on server)

Step 3: Dashboard Overview

Key iDRAC sections:

  • Server health status
  • Hardware inventory
  • Quick launch buttons
  • Alerts and logs

Launching Virtual Console (iDRAC)

  1. Navigate to: iDRAC Settings → Virtual Console
  2. Click "Launch Virtual Console"
  3. Accept Java/WebSocket certificate
  4. Virtual Console window opens

Keyboard Shortcuts:

  • Ctrl+Alt+Del → Send Ctrl+Alt+Del to server
  • Ctrl+Alt+Esc → Exit to BIOS/UEFI
  • Ctrl+Alt+F1 → Switch to text console

4.2 HP iLO Configuration

Accessing iLO Web Interface

Step 1: Open Web Browser

URL: https://<iLO-IP>

Example: https://192.168.1.101

Step 2: Login Credentials

Default Credentials (Factory Reset):

  • Username: Administrator
  • Password: (label on server or random)

iLO Virtual Media Setup

  1. Navigate to: Virtual Media → Virtual Media Launcher
  2. Select: CD/DVD-ROM
  3. Browse to: RHEL 9.5 ISO file
  4. Click: Connect CD/DVD
  5. Verify: Connected status shown

4.3 IPMI/KVM Access

IPMI over LAN Configuration

Via Command Line (ipmitool):

# Set IP address
ipmitool -I lanplus -H <BMC-IP> -U ADMIN -P 'password' set ipaddr 192.168.1.102

# Enable KVM
ipmitool -I lanplus -H <BMC-IP> -U ADMIN -P 'password' set sol enabled

# Activate SOL (Serial Over LAN)
ipmitool -I lanplus -H <BMC-IP> -U ADMIN -P 'password' sol activate

5. Mounting RHEL ISO via Virtual Media

5.1 Dell iDRAC Virtual Media

Method 1: Web Interface

  1. Login to iDRAC Web UI
  2. Navigate to: Virtual Media → Connected
  3. Click: "Map CD/DVD"
  4. Browse to: rhel-9.5-x86_64-dvd.iso
  5. Select: "Map Drive"
  6. Verify: Status shows "Connected"

Method 2: racadm Command Line

# Mount ISO using racadm
racadm -r 192.168.1.100 -u root -p calvin remoteimage -c
racadm -r 192.168.1.100 -u root -p calvin remoteimage -l //fileserver/share/rhel-9.5-x86_64-dvd.iso

# Check status
racadm -r 192.168.1.100 -u root -p calvin remoteimage -s

5.2 HP iLO Virtual Media

Method 1: iLO Web Interface

  1. Login to iLO web interface
  2. Navigate to: Virtual Media
  3. Click: "Launch Virtual Media Launcher"
  4. In new window:
    • Select: "CD/DVD" from dropdown
    • Click: "Browse" to find ISO
    • Click: "Connect"
    • Verify connection status

5.3 IPMI Virtual Media (Supermicro)

Via Web Interface

  1. Login to Supermicro IPMI web interface
  2. Navigate to: Virtual Media → Virtual Media (KVM/IP)
  3. Click: "CD-ROM Image"
  4. Browse and select RHEL ISO
  5. Click: "Mount"
  6. Status shows: "Mounted: rhel-9.5-x86_64-dvd.iso"

5.4 Virtual Media Best Practices

Performance Tips:
  • Use local network/file server for ISO (reduce latency)
  • Prefer 1 Gbps+ management network
  • Close other browser tabs during install
  • Disable browser caching for remote console
Reliability Tips:
  • Verify ISO file checksum (sha256sum) before mounting
  • Use Full DVD ISO, not Boot ISO (includes all packages)
  • Ensure stable power before starting install

6. BIOS/UEFI Boot Configuration

6.1 Accessing BIOS/UEFI Setup

Via Remote Console

  1. Open Virtual Console (iDRAC/iLO/KVM)
  2. During boot, press appropriate key:
    • Dell PowerEdge: F2 (or Ctrl+E for BIOS)
    • HP ProLiant: F9 (System Utilities)
    • Supermicro: Del (or F2)
  3. Navigate to: Boot Settings / BIOS Settings

Via Remote Boot Menu (One-Time)

  1. Open Virtual Console
  2. Reboot server (via remote power control)
  3. Press F11/F12 for Boot Menu
  4. Select: "Virtual CD/DVD" or "BIOS Boot Manager"

6.2 UEFI Boot Settings Configuration

BOOT ORDER (Top to Bottom):

  1. [✓] Virtual CD/DVD (first)
  2. [✓] Hard Disk / RAID Controller
  3. [ ] Network (PXE) - Disable unless needed for PXE install
  4. [ ] USB Device

UEFI SETTINGS:

  • Secure Boot: Disabled (for RHEL 9.5)
  • Boot Mode: UEFI (not Legacy)
  • CSM Support: Disabled

6.3 Dell-Specific BIOS Settings

iDRAC BIOS Configuration

  1. Boot into System BIOS (F2 during boot)
  2. Navigate: System BIOS → Boot Settings

Boot Settings:

  • Boot Mode: UEFI
  • Boot Sequence: [Virtual CD/DVD, Hard Disk]
  • Secure Boot: Disabled

6.4 HP-Specific BIOS Settings

iLO BIOS Configuration

  1. Boot into System Utilities (F9 during boot)
  2. Navigate: System Configuration → BIOS/Platform Configuration (RBSU)

BIOS Settings:

  • Boot Mode: UEFI
  • Boot Order: [Virtual Media, AHCI/RAID]
  • Virtualization Technology: Enabled

7. RAID Configuration

7.1 RAID Controller Access

Dell PERC (PowerEdge RAID Controller)

  1. During boot, press: Ctrl+R for PERC BIOS
  2. Or via iDRAC: Server Configuration → Storage Controller
  3. View: Physical Disks, Virtual Disks, Configuration

HP Smart Array

  1. During boot, press: F10 for Smart Array BIOS
  2. Or via iLO: Storage → Smart Array
  3. Configure: Arrays, Logical Drives, Spare Drives

7.2 RAID Configuration Options

RAID Level Min Disks Use Case Notes
RAID 0 2 Test/Dev No redundancy (avoid)
RAID 1 2 Boot/OS Mirroring, best for OS
RAID 5 3 General Good balance (deprecated)
RAID 6 4 Data Dual parity, high reliability
RAID 10 4 Database Performance + redundancy
Recommended for RHEL 9.5 Installation:
  • OS Disk: RAID 1 (two mirrored disks)
  • Data Disk: RAID 6 or RAID 10 depending on performance needs
  • Use Hot Spare: Yes (recommended for production)

7.3 Creating Virtual Disk (PERC Example)

  1. Press Ctrl+R during boot (PERC BIOS)
  2. Select: Controller (Slot 6: PERC H750)
  3. Press: F2 → Create New VD (Virtual Disk)

Create Virtual Disk:

  • Name: RHEL_BOOT
  • RAID Level: RAID 1
  • Physical Disks: [Disk 0:1, Disk 0:2] (Select 2)
  • Cache Policy: Read Ahead, Write Back
  • Initialize: Yes (quick init)

10. Enterprise Best Practices

10.1 Pre-Installation Best Practices

Documentation:

  • Complete server configuration worksheet before starting
  • Document IP addressing scheme and VLAN assignments
  • Record service tag and iDRAC/iLO IP addresses
  • Note DNS, NTP, and network proxy requirements

Software Preparation:

  • Download official RHEL 9.5 ISO from Red Hat Customer Portal
  • Verify ISO checksum (sha256sum): 
    $ sha256sum rhel-9.5-x86_64-dvd.iso
abc123... rhel-9.5-x86_64-dvd.iso
  • Use DVD ISO, not Boot ISO (for offline installation)

Hardware Verification:

  • Update firmware (iDRAC/iLO, PERC, BIOS) before install
  • Test hardware health via remote management
  • Verify RAID configuration meets data protection requirements
  • Ensure server has current IPMI/BMC firmware

10.2 Post-Installation Best Practices

# 1. System Registration
subscription-manager register --username=<username> --password=<password>
subscription-manager attach --pool=<pool-id>
subscription-manager repos --enable=rhel-9-for-x86_64-baseos-eus

# 2. Update System
dnf update -y
reboot

# 3. Configure Firewalld
firewall-cmd --permanent --add-service=ssh
firewall-cmd --permanent --add-service=https
firewall-cmd --reload

# 4. Enable RHEL Services
systemctl enable --now rsyslog
systemctl enable --now chronyd
systemctl enable --now firewalld

# 5. Install Additional Enterprise Tools
dnf install -y wget curl net-tools bind-utils strace
dnf install -y cockpit-system cockpit-storaged
systemctl enable --now cockpit.socket

10.3 Security Hardening Checklist

  • Root Account
    • Disable direct root SSH login
    • Use sudo for administrative tasks
  • SSH Hardening
    • Disable password authentication (use keys)
    • Disable root login
  • Firewall
    • Enable firewalld
    • Restrict to management network
  • SELinux
    • Keep enabled (Enforcing mode)
  • Updates
    • Subscribe to errata notifications
    • Schedule regular updates

11. Common Mistakes in Remote Installs

11.1 Virtual Media Issues

MISTAKE: ISO mounted but not detected as boot device

CAUSES:

  • Boot order not set to virtual CD/DVD first
  • Virtual media mount not "Connected" (just "Mapped")
  • Browser caching old iDRAC/iLO session
  • UEFI boot mode vs Legacy boot mode mismatch

SOLUTION:

  • Reboot server after mounting ISO
  • Verify boot order in BIOS before boot
  • Clear browser cache or use Incognito/Private mode
  • Verify boot mode in BIOS matches ISO (UEFI for RHEL 9.x)
MISTAKE: Virtual media disconnects during installation

CAUSES:

  • Browser timeout/session expiration
  • Network interruption
  • Management network switch port flapping

SOLUTION:

  • Use stable management network
  • Extend session timeout in iDRAC/iLO settings
  • Do not browse away from remote console during install

11.2 Boot and Partitioning Mistakes

MISTAKE: "No operating system found" after reboot

CAUSES:

  • Boot order reverted after installation
  • RAID configuration lost/reset
  • Boot loader not installed correctly
  • Disk not properly partitioned for boot

SOLUTION:

  • Reconfigure boot order in BIOS after install
  • Verify RAID Virtual Disk is still "Online"
  • Reinstall bootloader to MBR/EFI partition
MISTAKE: Wrong partition scheme for UEFI

CAUSES:

  • Missing EFI System Partition (ESP)
  • ESP not formatted as FAT (not EFI)

SOLUTION:

  • Create /boot/efi as 600MB EFI System Partition
  • Ensure File System is "EFI System Partition" (FAT)
  • Boot in UEFI mode in BIOS settings

11.3 Network and Configuration Mistakes

MISTAKE: No network connectivity after installation

CAUSES:

  • Wrong VLAN configured
  • Gateway not set
  • DNS not configured
  • Network interface not enabled on boot

SOLUTION:

  • Confirm VLAN ID with network team
  • Add default gateway: ip route add default via <gateway>
  • Add DNS: /etc/resolv.conf or via NetworkManager
  • Enable interface: nmcli connection up <interface-name>
MISTAKE: Cannot register with Red Hat Subscription

CAUSES:

  • Wrong credentials
  • No internet connectivity (requires proxy)
  • Subscription pool already consumed
  • System time incorrect (certificates rejected)

SOLUTION:

  • Verify credentials at access.redhat.com
  • Configure proxy in /etc/rhsm/rhsm.conf if needed
  • Check available subscriptions: subscription-manager list
  • Sync NTP: timedatectl set-ntp true

12. Troubleshooting Guide

12.1 Console Not Loading Issues

PROBLEM: Remote Console (KVM) won't launch

DIAGNOSTIC STEPS:

  1. Check Java/WebSocket plugin status
  2. Verify network connectivity - Ping iDRAC/iLO IP
  3. Check license (iDRAC requires Enterprise, iLO requires Advanced)
  4. Try alternate browser
  5. Check security software (proxies may block)

COMMON ERROR MESSAGES:

  • "Unable to launch virtual console" → Solution: Clear browser cache, use HTTPS
  • "Java plugin not available" → Solution: Use HTML5 console or install Java
  • "Session already in use" → Solution: Disconnect other sessions

12.2 ISO Mounting Issues

PROBLEM: ISO mounted but system doesn't boot from it

DIAGNOSTIC STEPS:

  1. Verify boot order in BIOS
  2. Check virtual media status - verify "Connected" not just "Mapped"
  3. Try different mounting method - use local file instead of network URL
  4. Verify boot mode (UEFI vs Legacy)

PROBLEM: Installation freezes at "Loading initrd"

CAUSES:

  • Corrupted ISO download
  • Slow network causing timeout

SOLUTION:

  • Verify ISO checksum
  • Use local file server for ISO
  • Re-download ISO from Red Hat

12.3 Boot Failure Issues

PROBLEM: System boots to "dracut:/#" shell (initramfs emergency)

CAUSES:

  • Missing drivers (RAID controller not recognized)
  • Disk configuration changed
  • Corrupted initramfs

SOLUTION:

  • Boot with "rhel_debug" option for diagnostics
  • Check: lsblk, lvm pvs, cat /proc/mounts
  • Rebuild initramfs: dracut -f

PROBLEM: "No bootable device found" after installation

DIAGNOSTIC STEPS:

  1. Check RAID configuration (is virtual disk online?)
  2. Check boot order in BIOS (did it reset?)
  3. Check if boot loader installed
  4. Verify UEFI boot entry: efibootmgr -v

12.4 Anaconda Installer Issues

PROBLEM: "Error checking storage configuration"

CAUSES:

  • Disk not properly initialized
  • RAID configuration not detected

SOLUTION:

  • Press Ctrl+Alt+F2 to access shell
  • Run: lsblk, lspci, cat /proc/partitions

PROBLEM: Package installation fails ("Package not found")

CAUSES:

  • ISO not complete (using Boot ISO instead of DVD)

SOLUTION:

  • Use full DVD ISO

PROBLEM: "Out of memory" during installation

CAUSES:

  • Less than 2GB RAM in system

SOLUTION:

  • Minimum 2GB RAM required, 4GB+ recommended
  • At installation boot: add inst.ram=4096

Completion Checklist

PRE-INSTALLATION:

  • RHEL 9.5 ISO downloaded and checksum verified
  • iDRAC/iLO/IPMI access verified and credentials ready
  • Virtual media successfully mounted
  • Boot order configured to Virtual CD first
  • RAID configuration completed (if applicable)

DURING INSTALLATION:

  • Language selected (English recommended)
  • Disk partitioning completed with LVM
  • Network configured with correct IP/gateway/DNS
  • Software selection matches requirements
  • Root password set (strong password)
  • User account created (if needed)
  • Installation completed without errors

POST-INSTALLATION:

  • Server rebooted successfully
  • Login works with created credentials
  • Network connectivity confirmed
  • SELinux in Enforcing mode
  • Firewall enabled and configured
  • System registered with Red Hat
  • All packages updated (dnf update)
  • Services enabled (rsyslog, chronyd, firewalld)
