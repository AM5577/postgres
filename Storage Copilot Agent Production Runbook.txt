1. OVERVIEW & PREREQUISITES
--------------------------------------------------------------------
This agent compares "before" and "after" storage configuration
snapshots (CSV files) for a storage array, cluster, or pool, and
produces a structured report covering configuration changes,
performance impact, capacity impact, drift, risk, and
recommendations.

Before you start, confirm you have:
[ ] Microsoft 365 Copilot Studio Premium (or Copilot Studio
    standalone with Premium/Pay-as-you-go add-on)
[ ] Power Platform environment (Dataverse enabled)
[ ] Power Automate Premium connectors enabled
[ ] Power BI Pro or Premium workspace
[ ] Admin access to Microsoft Teams (for deployment)
[ ] Sample snapshot CSV files (e.g., Device_A_2026-06-01.csv,
    Device_A_2026-07-01.csv)
[ ] A security group of approved storage administrators

4. STEP-BY-STEP BUILD IN COPILOT STUDIO
--------------------------------------------------------------------
STEP 4.1 — Create the Agent
  1. Go to https://copilotstudio.microsoft.com
  2. Select your target environment (must have Dataverse enabled).
  3. Click "+ Create" > "New agent" > "Skip to configure".
  4. Name: "Storage Change Impact Intelligence Copilot"
  5. Description: paste a short version of the mission statement
     (see Appendix for full text).
  6. Choose "Conversational" agent type.

STEP 4.2 — Set Instructions (System Prompt)
  1. Open the "Overview" tab of your agent.
  2. In the "Instructions" field, paste the full system prompt
     from Section 17 (Appendix) — this defines role, inputs, data
     fields, analysis logic, output format, and escalation rules.
  3. Save and click "Test" in the side panel to validate tone and
     structure with a sample question.

STEP 4.3 — Enable Generative Orchestration
  1. Go to Settings > Orchestration.
  2. Enable "Generative AI orchestration" so the agent can
     dynamically decide which topic/action to call based on user
     intent (e.g., "compare snapshots" vs "show risk only").

STEP 4.4 — Connect Knowledge Sources (Optional but Recommended)
  1. Go to "Knowledge" tab > "+ Add knowledge".
  2. Add your enterprise baseline standards document (RAID
     policies, cache policy standards, replication standards) so
     the agent can perform accurate drift detection.
  3. Add historical incident reports if available, to improve AI
     Insights quality.

6. TOPICS & CONVERSATION DESIGN
--------------------------------------------------------------------
Create these Topics in Copilot Studio (Topics tab):

TOPIC 1: "Compare Snapshots"
  Trigger phrases: "compare snapshots", "what changed",
  "analyze device", "compare before and after"
  Flow: Ask for Device Name > Device Type > Before Snapshot file >
  After Snapshot file > Call Power Automate flow
  "GetSnapshotComparison" > Return structured 8-section report.

TOPIC 2: "Risk Check Only"
  Trigger phrases: "what is the risk", "risk assessment for [device]"
  Flow: Retrieve latest comparison from Dataverse > Return only
  Risk Assessment + Escalation status.

TOPIC 3: "Drift Check"
  Trigger phrases: "configuration drift", "is this compliant"
  Flow: Compare config fields to baseline table in Dataverse >
  Return drift status only.

TOPIC 4: "Escalation"
  System topic, triggered automatically when Risk = High/Critical,
  Utilization > 90%, Latency increase > 30%, redundancy reduced, or
  replication disabled.
  Flow: Post adaptive card to Teams channel + send email via
  Power Automate "EscalationAlert" flow.

TOPIC 5: "Fallback/Help"
  Standard fallback topic explaining what the agent can do and
  prompting for Device Name + snapshot files.


7. POWER AUTOMATE FLOWS (DATA PLUMBING)
--------------------------------------------------------------------
FLOW A — "Snapshot Ingestion"
  Trigger: File created in SharePoint/OneDrive folder OR manual
  upload via Copilot Studio file upload control.
  Actions:
    1. Parse CSV (use "Parse a Document" or a CSV-to-JSON action).
    2. Validate required columns exist (RAID Policy, Cache Policy,
       IOPS, Latency, Throughput, Utilization %, etc.).
    3. Insert/Update rows into Dataverse table "StorageSnapshots".
    4. Return success/failure to the calling agent.

FLOW B — "GetSnapshotComparison"
  Trigger: Called from Copilot Studio Topic 1.
  Actions:
    1. Query Dataverse for Before and After snapshot rows matching
       Device Name + dates.
    2. Compute percentage changes for IOPS, Latency, Throughput,
       Cache Hit Ratio using the formula:
       ((After - Before) / Before) x 100
    3. Compare configuration fields row by row; build a list of
       changed fields.
    4. Compare against "BaselineStandards" Dataverse table; flag
       drift level (Compliant / Minor / Critical).
    5. Apply risk scoring logic (Section 14 rules).
    6. Return a JSON payload back to Copilot Studio matching the
       8-section output schema.

FLOW C — "EscalationAlert"
  Trigger: Called automatically when Flow B returns Risk = High or
  Critical, or any hard trigger condition is met.
  Actions:
    1. Post adaptive card to a designated Teams channel.
    2. Send email to the storage on-call distribution list.
    3. Log the escalation event in Dataverse table
       "EscalationLog" (for metrics tracking).

FLOW D — "PowerBIRefresh"
  Trigger: Scheduled (e.g., every night) or triggered after Flow B
  completes.
  Actions:
    1. Call Power BI REST API "Refresh Dataset" action.
    2. Confirm refresh status; log failures to Dataverse.


8. DATAVERSE SETUP (DATA STORAGE)
--------------------------------------------------------------------
Create these tables:

TABLE: StorageSnapshots
  Columns: DeviceName, DeviceType, SnapshotDate, RAIDPolicy,
  CachePolicy, ReplicationSettings, TieringPolicy, HostMappings,
  PoolAllocation, DeduplicationSettings, CompressionSettings,
  IOPS, Latency, Throughput, CacheHitRatio, ReadWriteRatio,
  UtilizationPct, AvailableCapacity, UsedCapacity, GrowthRate

TABLE: BaselineStandards
  Columns: DeviceType, StandardRAIDPolicy, StandardCachePolicy,
  StandardReplication, StandardTieringPolicy, EffectiveDate

TABLE: ComparisonResults
  Columns: DeviceName, ComparisonDate, ExecutiveSummary,
  ConfigChangesJSON, PerformanceImpactJSON, CapacityImpactJSON,
  DriftStatus, RiskLevel, AIInsights, RecommendedActions

TABLE: EscalationLog
  Columns: DeviceName, EscalationDate, RiskLevel, TriggerReason,
  NotifiedChannel, Acknowledged (Yes/No)

Set security roles so only the Storage Admin security group can
write to these tables; read access can be broader if needed for
reporting.


9. POWER BI DASHBOARD INTEGRATION
--------------------------------------------------------------------
  1. Connect Power BI Desktop to the Dataverse tables above via the
     Dataverse connector.
  2. Build visuals:
     - Trend line: IOPS/Latency/Throughput over time per device
     - Utilization % gauge with red/yellow/green thresholds
     - Drift status by device (bar chart)
     - Risk level heatmap across all devices
     - Escalation count over time
  3. Publish to a Power BI workspace shared with storage admins.
  4. Set a scheduled refresh (or trigger via Flow D above).
  5. Optionally embed the Power BI report directly inside the
     Copilot Studio agent's response using the Power BI card.


10. SECURITY, PERMISSIONS & DATA LOSS PREVENTION
--------------------------------------------------------------------
  - Apply Data Loss Prevention (DLP) policies in Power Platform
    Admin Center restricting which connectors this environment can
    use (block social/consumer connectors; allow only Dataverse,
    Outlook, Teams, Power BI, SharePoint, HTTP-with-approval).
  - Restrict agent publishing rights to a small admin group.
  - Enable Microsoft Entra ID authentication on the Copilot Studio
    agent (no anonymous access) since storage configs are sensitive.
  - Apply row-level security in Dataverse if multiple business
    units share the same environment.
  - Enable audit logging (Microsoft Purview / Dataverse auditing)
    on all four tables.
  - Review the agent's generative AI "moderation" and "content
    safety" settings in Copilot Studio to prevent leaking one
    device's data into another device's conversation context.


11. TESTING PLAN
--------------------------------------------------------------------
  Unit Tests:
   [ ] Upload two snapshots with no changes -> agent reports "no
       configuration changes, Low risk"
   [ ] Upload snapshots with RAID change only -> agent flags config
       change and appropriate drift status
   [ ] Upload snapshots with latency increase > 30% -> agent
       triggers escalation flow automatically
   [ ] Upload snapshots with utilization > 90% -> escalation fires
   [ ] Ask "what is the risk for Device A" without running a full
       comparison first -> agent gracefully asks user to run a
       comparison or pulls last saved result from Dataverse

  Integration Tests:
   [ ] Verify Power Automate flow error handling (missing file,
       malformed CSV, missing baseline record)
   [ ] Verify Teams adaptive card renders correctly for escalations
   [ ] Verify Power BI dashboard refresh completes end to end

  User Acceptance Testing (UAT):
   [ ] Have 3-5 real storage admins run real snapshot comparisons
       and confirm report accuracy and clarity
   [ ] Collect feedback on wording of AI Insights section

14. ESCALATION & ALERTING RULES
--------------------------------------------------------------------
Escalate immediately (auto-trigger Flow C) if ANY of the following:
  - Overall Risk = High or Critical
  - Utilization exceeds 90%
  - Latency increases more than 30%
  - Redundancy is reduced (e.g., RAID level lowered, mirrors
    removed)
  - Replication is disabled or downgraded

Escalation channel: Teams channel "#storage-ops-alerts" + email
distribution list "storage-oncall@yourcompany.com" (replace with
your actual list).

  - Review baseline standards table quarterly; update as enterprise

  - Version the system prompt (keep a changelog) — any change to

  - Re-run the Testing Plan (Section 11) after any Copilot Studio

  - Rotate/reauthorize Power Automate connections per your
    organization's credential rotation policy.


16. TROUBLESHOOTING GUIDE
--------------------------------------------------------------------
Problem: Agent doesn't detect configuration changes correctly
  Check: CSV column headers match exactly what Flow B expects;
  check Dataverse row insertion succeeded in Flow A.

Problem: Escalation not firing on high risk
  Check: Flow B's risk output value matches exactly what Flow C's
  trigger condition expects (case-sensitive string match issues are
  common).

Problem: Power BI dashboard shows stale data
  Check: Flow D schedule/trigger; check Power BI dataset refresh
  history for errors; check Dataverse connector credentials.

Problem: Agent gives inconsistent report formatting
  Check: System prompt Output Format section hasn't been
  accidentally edited; check generative orchestration isn't
  overriding the locked topic flow.

Problem: Users get "access denied"
  Check: Entra ID group membership; Dataverse security role
  assignment; DLP policy blocking a connector.
