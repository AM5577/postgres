1. Introduction & Overview

Purpose

This guide provides step-by-step instructions for installing Red Hat Enterprise Linux 9.5 on bare metal servers using remote console technologies (iDRAC, iLO, or IPMI). This methodology is essential for:

  • Datacenter deployments where physical server access is impractical
  • Remote/hybrid work environments requiring out-of-band management
  • Enterprise standardization of OS deployment procedures
  • Disaster recovery scenarios needing rapid OS reinstallation

Target Audience

  • System Administrators
  • DevOps Engineers
  • Infrastructure Architects
  • RHCA-Certified Professionals

RHEL 9.5 Key Features

Feature Description
UEFI Secure Boot Enhanced security with signed bootloaders
LVM Thin Provisioning Efficient storage allocation
Network Stack Improved NetworkManager with keyfile support
SELinux Enforcing mode by default
Firmware Modern hardware support (AMD Genoa, Intel Sapphire Rapids)

10. Reboot and Validation

Post-Installation Checklist

1. Verify Network Connectivity

# Check IP address
ip addr show

# Test connectivity
ping -c 4 8.8.8.8

# Verify DNS resolution
nslookup google.com

2. Check Disk Partitioning

# List block devices
lsblk

# Show mounted filesystems
df -h

# Show LVM volumes
pvs
vgs
lvs

3. Verify System Services

# Check critical services status
systemctl status sshd
systemctl status firewalld
systemctl status rsyslog
systemctl status NetworkManager

4. Verify SELinux Status

# Check SELinux status
getenforce
# Output: Enforcing

# Check SELinux context
sestatus

5. Register with Red Hat (If Required)

# Register the system
subscription-manager register --username=your_username --password=your_password

# Attach subscription
subscription-manager attach --auto

# Check subscription status
subscription-manager status

6. Update System

# Check for updates
dnf check-update

# Apply updates (if available)
sudo dnf update -y

# Reboot if kernel updated
sudo reboot

Post-Installation Security Hardening

# 1. Disable unnecessary services
sudo systemctl disable --now cups
sudo systemctl disable --now avahi-daemon

# 2. Configure firewall
sudo firewall-cmd --permanent --add-service=ssh

sudo firewall-cmd --reload

# 3. Enable audit logging
sudo auditctl -w /etc/passwd -p wa -k user_mod
sudo auditctl -w /etc/shadow -p wa -k shadow_mod

# 1. Tune kernel parameters (sysctl)
sudo sysctl -w net.core.somaxconn=65535

sudo sysctl -w vm.swappiness=10

# 2. Configure tuned (performance profile)

12. Common Mistakes in Remote Installs

Mistake 1: Virtual Media Not Properly Connected

Symptom: Server boots from hard drive instead of ISO

Solution:

  • ✅ Ensure ISO is mounted as "Connected" not just "Inserted"
  • ✅ Check that "Connect Virtual Media at Boot" is enabled
  • ✅ Verify boot order has Virtual CD/DVD at position 1
  • ✅ Use "One-Time Boot" to force virtual media for this boot

Mistake 2: ISO Not Accessible via Network

Symptom: "Unable to retrieve installation image"

Solution:

  • ✅ Verify HTTP/HTTPS server is running and accessible
  • ✅ Test URL from another machine: curl -I http://[IP]/rhel-9.5.iso
  • ✅ Check firewall on ISO host
  • ✅ Ensure no authentication required for ISO download

Mistake 3: Incorrect Boot Mode (UEFI vs BIOS)

Symptom: "No operating system found" after installation

Solution:

  • ✅ Verify boot mode matches installation mode
  • ✅ Check UEFI settings: Boot Mode should be "UEFI Only"
  • ✅ Ensure EFI partition (/boot/efi) is created for UEFI

Mistake 4: Partition Size Misconfiguration

Symptom: /var or /tmp fills up, system unusable

Solution:

  • ✅ Plan partition sizes based on workload
  • ✅ Allocate more space to /var for logs and containers
  • ✅ Use LVM for flexibility to resize partitions later

Mistake 5: Network Configuration Errors

Symptom: No network connectivity after installation

Solution:

  • ✅ Verify static IP configuration is correct
  • ✅ Check gateway and netmask
  • ✅ Ensure DNS servers are reachable
  • ✅ Verify NetworkManager is enabled and running

13. Troubleshooting

Issue 1: Remote Console Not Loading

Symptoms: "Unable to launch virtual console", Connection timeout

Resolution Steps:
  1. Check browser compatibility - Use Chrome/Edge for iDRAC, Firefox for iLO
  2. Enable Java runtime (if using Java-based console)
  3. Clear browser cache and cookies for BMC IP
  4. Check network connectivity: ping [BMC-IP], telnet [BMC-IP] 443
  5. Verify BMC firmware is up to date

Issue 2: ISO Mount Issues

Symptoms: "No media found", "Unable to connect to image"

Resolution Steps:
  1. Verify HTTP Server: curl -I http://192.168.1.50:8000/rhel-9.5.iso
  2. Check ISO File Permissions: chmod 644 /opt/rhel-iso/rhel-9.5.iso
  3. Try Alternative Methods: CIFS/SMB share instead of HTTP

Issue 3: Boot Failure After Installation

Symptoms: "No bootable device found", Boot loop

Resolution Steps:
  1. Check boot order in BIOS/UEFI
  2. Verify bootloader installation (from rescue mode)
  3. Recreate initramfs: dracut --force
  4. Reinstall GRUB: grub2-install /dev/sda

Issue 4: Network Not Working After Install

Symptoms: No IP address assigned, Cannot ping gateway

Resolution Steps: 
# 1. Check NetworkManager status
systemctl status NetworkManager
nmcli device status

# 2. View connection details
nmcli connection show

# 3. Manually bring up interface
nmcli connection up ens160

# 4. Test DNS resolution
nslookup google.com 8.8.8.8

Issue 5: Installation Freezes/Hangs

Common Causes & Fixes:
  1. Insufficient Memory: Minimum 2 GB RAM required; 4 GB recommended
  2. Disk Issues: Check SMART status, verify RAID controller
  3. ISO Media Corruption: Verify SHA256 checksum
  4. BMC Performance: Reduce remote console resolution/quality
  5. Virtual Media Driver Issues: Try different method (CIFS vs HTTP)
