SECTION 5: HARDWARE SIZING RECOMMENDATIONS
================================================================================

5.1 CONTROL PLANE NODES (MINIMUM PRODUCTION)
---------------------------------------------
  vCPUs  : 4 cores (8 recommended for >50 managed clusters)
  RAM    : 16 GB (32 GB recommended for >50 managed clusters)
  Disk   : 
    OS   : 50 GB (SSD/NVMe preferred)
    etcd : 20 GB dedicated NVMe (CRITICAL - high IOPS required)
    Logs : 20 GB
  Network: 1 Gbps (10 Gbps recommended for production)

  CAUTION: etcd is extremely I/O sensitive. Use NVMe SSDs for etcd data.
  Slow disk (HDD or slow SSD) is the #1 cause of etcd instability.
  etcd disk benchmarks should show < 10ms write latency (use fio to verify).

  FIO BENCHMARK COMMAND (run on etcd disk before installation):
  fio --rw=write --ioengine=sync --fdatasync=1 --directory=/var/lib/rancher/rke2/server/db \
      --size=22m --bs=2300 --name=etcd-disk-benchmark

  Interpret: 99th percentile fsync latency < 10ms = acceptable
             > 25ms = DO NOT use for etcd

5.2 WORKER NODES (MINIMUM PRODUCTION)
--------------------------------------
  vCPUs  : 8 cores (scale based on workload)
  RAM    : 32 GB (scale based on workload)
  Disk   :
    OS   : 100 GB (SSD preferred)
    Data : 200 GB+ (for container storage, scale to workload)
  Network: 1 Gbps (10 Gbps for high-throughput workloads)

  NOTE: Worker node sizing depends entirely on workload. These are minimums
  for running Rancher system components + typical enterprise apps.

5.3 LOAD BALANCER NODE
-----------------------
  vCPUs  : 2 cores
  RAM    : 4 GB
  Disk   : 30 GB
  Network: 1 Gbps (must handle total cluster ingress bandwidth)
  HA     : Configure 2x LB nodes with keepalived for VIP failover

5.4 STORAGE SIZING
-------------------
  etcd   : 8 GB typical, 20 GB with generous headroom (never let etcd DB > 8 GB)
  Rancher: 10 GB PVC for Rancher itself (audit logs, etc.)
  Monitoring: 50-100 GB for Prometheus (7-30 day retention)
  Logging: 100-500 GB for log storage (varies by retention policy)
  App PVCs: Size per application requirements

6.2 DISABLE SWAP (KUBERNETES REQUIREMENT)
-------------------------------------------
# Disable swap immediately
swapoff -a

# Permanently disable swap (comment out swap lines)
sed -i '/ swap / s/^/#/' /etc/fstab

# Verify swap is disabled
free -h | grep Swap

# WHY: Kubernetes does not support swap by default. Swap causes unpredictable
# scheduling behavior and degrades performance. kubelet will fail to start
# if swap is enabled (unless explicitly configured, which is NOT recommended
# for production).

6.3 DISABLE UNNECESSARY SERVICES
-----------------------------------
# Disable services not needed on Kubernetes nodes
systemctl disable --now \
  postfix \
  cups \
  avahi-daemon \
  bluetooth \
  rpcbind

# Check running services
systemctl list-units --state=running --type=service

# WHY: Principle of least privilege - disable services that are not required.
# Reduces attack surface and avoids port conflicts.

6.4 SET HOSTNAME AND UPDATE /etc/hosts
----------------------------------------
# On each node, set appropriate hostname
hostnamectl set-hostname cp-node-01.example.com   # Change per node

6.5 CONFIGURE SSH HARDENING
------------------------------
# Backup original sshd_config
cp /etc/ssh/sshd_config /etc/ssh/sshd_config.bak

# Apply hardened SSH configuration
cat > /etc/ssh/sshd_config << 'EOF'
# Production SSH Configuration - Hardened
Protocol 2
Port 22
AddressFamily inet
ListenAddress 0.0.0.0

# Authentication
PermitRootLogin no
PasswordAuthentication no
PubkeyAuthentication yes
AuthorizedKeysFile .ssh/authorized_keys
PermitEmptyPasswords no
ChallengeResponseAuthentication no
UsePAM yes

# Session
LoginGraceTime 30
MaxAuthTries 3
MaxSessions 5
ClientAliveInterval 300
ClientAliveCountMax 2
TCPKeepAlive no

# Security
X11Forwarding no
AllowAgentForwarding no
AllowTcpForwarding no
PermitTunnel no
Banner /etc/ssh/banner
PrintLastLog yes

# Ciphers and MACs (Strong only)
Ciphers chacha20-poly1305@openssh.com,aes256-gcm@openssh.com,aes128-gcm@openssh.com
MACs hmac-sha2-512-etm@openssh.com,hmac-sha2-256-etm@openssh.com
KexAlgorithms curve25519-sha256,curve25519-sha256@libssh.org,diffie-hellman-group16-sha512

# Logging
SyslogFacility AUTH
LogLevel VERBOSE

# Allow only specific users/groups
AllowGroups sshusers wheel

# Subsystem
Subsystem sftp /usr/lib/openssh/sftp-server
EOF

# Create SSH banner
cat > /etc/ssh/banner << 'EOF'
*******************************************************************************
                          AUTHORIZED ACCESS ONLY
  This system is restricted to authorized users for business purposes only.
  Unauthorized access is prohibited and will be prosecuted to the full extent
  of applicable law. All activities are monitored and logged.
*******************************************************************************
EOF

# Restart SSH (do NOT close existing sessions until verified)
systemctl restart sshd

# WHY: Default SSH config allows many insecure options. We disable root login,
# enforce key-based auth, restrict ciphers to strong algorithms, and limit
# forwarding to prevent SSH tunneling abuse.

6.6 CONFIGURE AUDIT LOGGING
-----------------------------
# Install auditd
dnf install -y audit audispd-plugins

# Deploy comprehensive audit rules
cat > /etc/audit/rules.d/99-kubernetes.rules << 'EOF'
# Delete all existing rules
-D

# Buffer size
-b 8192

# Failure mode (2=panic, 1=printk)
-f 1

# Monitor authentication
-w /etc/passwd -p wa -k identity
-w /etc/group -p wa -k identity
-w /etc/shadow -p wa -k identity
-w /etc/sudoers -p wa -k identity
-w /etc/sudoers.d/ -p wa -k identity

# Monitor SSH
-w /etc/ssh/sshd_config -p wa -k sshd

# Monitor Kubernetes config files
-w /etc/rancher/ -p wa -k rancher
-w /var/lib/rancher/ -p rwxa -k rancher
-w /etc/kubernetes/ -p wa -k kubernetes

# Monitor privileged commands
-a always,exit -F arch=b64 -S execve -F euid=0 -k root-commands
-a always,exit -F arch=b64 -S execve -F egid=0 -k root-commands

# Monitor sudo usage
-w /usr/bin/sudo -p x -k sudo
-w /usr/bin/su -p x -k sudo

# System calls
-a always,exit -F arch=b64 -S adjtimex,settimeofday -k time-change
-a always,exit -F arch=b64 -S clock_settime -k time-change
-a always,exit -F arch=b64 -S sethostname,setdomainname -k system-locale

# Network changes
-a always,exit -F arch=b64 -S sethostname,setdomainname -k network_modifications

# Successful/failed logins
-w /var/log/lastlog -p wa -k logins
-w /var/run/faillock/ -p wa -k logins

# Container runtime monitoring
-w /usr/bin/runc -p x -k container-runtime
-w /usr/bin/containerd -p x -k container-runtime
EOF

# Enable and start auditd
systemctl enable --now auditd
augenrules --load

# WHY: Audit logging provides forensic evidence for security incidents,
# compliance with standards like PCI-DSS, SOC2, ISO27001. Kubernetes-specific
# rules capture changes to cluster configuration files.

7.1 REQUIRED KERNEL MODULES
------------------------------
# Load required kernel modules immediately
modprobe br_netfilter

modprobe nf_conntrack_ipv4

# Make modules persistent across reboots
cat > /etc/modules-load.d/rke2.conf << 'EOF'
# RKE2 Required Kernel Modules
br_netfilter
overlay
ip_tables
ip6_tables
nf_conntrack
nf_conntrack_ipv4
nf_conntrack_ipv6
xt_REDIRECT
xt_owner
xt_statistic
EOF

# Verify modules are loaded
lsmod | grep -E 'br_netfilter|overlay|ip_tables|nf_conntrack'

# WHY: 
# - br_netfilter: Required for iptables to see bridged network traffic
# - overlay: Required for container filesystem layers (overlayfs)
# - ip_tables/ip6_tables: Required for kube-proxy networking rules
# - nf_conntrack: Required for connection tracking (load balancing, NAT)

7.2 SYSCTL KERNEL PARAMETERS
-------------------------------
cat > /etc/sysctl.d/99-kubernetes.conf << 'EOF'
# =============================================================================
# Kubernetes / RKE2 Production Kernel Parameters
# =============================================================================

# --- Networking ---
# Enable IPv4 forwarding (REQUIRED for pod networking)
net.ipv4.ip_forward = 1

# Enable IPv6 forwarding (if using IPv6)
net.ipv6.conf.all.forwarding = 1

# Enable bridged traffic iptables (REQUIRED - br_netfilter must be loaded first)
net.bridge.bridge-nf-call-iptables = 1
net.bridge.bridge-nf-call-ip6tables = 1
net.bridge.bridge-nf-call-arptables = 1

# Increase network buffer sizes for high-throughput
net.core.rmem_max = 134217728
net.core.wmem_max = 134217728
net.core.rmem_default = 31457280
net.core.wmem_default = 31457280
net.core.optmem_max = 25165824
net.core.somaxconn = 65535
net.core.netdev_max_backlog = 300000

# TCP optimizations
net.ipv4.tcp_rmem = 4096 87380 134217728
net.ipv4.tcp_wmem = 4096 65536 134217728
net.ipv4.tcp_mem = 786432 1048576 26777216
net.ipv4.tcp_max_syn_backlog = 8192
net.ipv4.tcp_slow_start_after_idle = 0
net.ipv4.tcp_fin_timeout = 15
net.ipv4.tcp_keepalive_time = 300
net.ipv4.tcp_keepalive_probes = 5
net.ipv4.tcp_keepalive_intvl = 15
net.ipv4.tcp_tw_reuse = 1

# UDP settings
net.ipv4.udp_rmem_min = 8192
net.ipv4.udp_wmem_min = 8192

# Conntrack table size (increase for large clusters)
net.netfilter.nf_conntrack_max = 1048576
net.nf_conntrack_max = 1048576

# Reverse path filter (0=disabled, 1=strict, 2=loose)
# Set to 0 for multi-homed nodes or specific networking scenarios
net.ipv4.conf.all.rp_filter = 0
net.ipv4.conf.default.rp_filter = 0

# --- File System ---
# Increase inotify limits for container monitoring
fs.inotify.max_user_watches = 1048576
fs.inotify.max_user_instances = 8192

# Increase file descriptors
fs.file-max = 2097152
fs.nr_open = 2097152

# --- Virtual Memory ---
# Reduce swappiness (should be 0 since swap is disabled, but set explicitly)
vm.swappiness = 0

# Dirty memory settings to avoid I/O spikes
vm.dirty_ratio = 5
vm.dirty_background_ratio = 2
vm.dirty_expire_centisecs = 1000

# --- Kernel ---
# Kernel panic on OOM (1=panic, 0=print only)
kernel.panic = 10

11.4 KUBERNETES AUDIT POLICY
------------------------------
# Create comprehensive audit policy (deploy on ALL control plane nodes)
cat > /etc/rancher/rke2/audit-policy.yaml << 'EOF'
apiVersion: audit.k8s.io/v1
kind: Policy
# Do not generate audit events for requests to these non-resource URLs
omitStages:
  - "RequestReceived"
rules:
  # Log pod changes at RequestResponse level
  - level: RequestResponse
    resources:
    - group: ""
      resources: ["pods", "pods/exec", "pods/attach", "pods/portforward"]

  # Log secrets at Metadata level (do NOT log request/response body for secrets!)
  - level: Metadata
    resources:
    - group: ""
      resources: ["secrets", "configmaps", "serviceaccounts"]

  # Log authentication decisions
  - level: Metadata
    nonResourceURLs:
    - /api*
    - /version

  # Log RBAC changes at RequestResponse
  - level: RequestResponse
    resources:
    - group: "rbac.authorization.k8s.io"
      resources: ["clusterroles", "clusterrolebindings", "roles", "rolebindings"]

  # Log persistent volume changes
  - level: RequestResponse
    resources:
    - group: ""
      resources: ["persistentvolumes", "persistentvolumeclaims"]

  # Log network policy changes
  - level: RequestResponse
    resources:
    - group: "networking.k8s.io"
      resources: ["networkpolicies"]

  # Log namespace changes
  - level: RequestResponse
    resources:
    - group: ""
      resources: ["namespaces"]

  # Default: log metadata for everything else
  - level: Metadata
    omitStages:
    - "RequestReceived"
EOF

# WHY: Kubernetes audit logs provide visibility into who did what in the cluster.
# Critical for compliance (SOC2, PCI-DSS) and incident response.
# We log secrets at Metadata only (not RequestResponse) to avoid leaking
# secret values to audit log storage.

15.2 PRIVATE CA CERTIFICATE SETUP
------------------------------------
# Option A: Create your own CA with OpenSSL
# --- Create Root CA ---
mkdir -p /opt/pki/{root-ca,intermediate-ca,certs}
cd /opt/pki/root-ca

# Create root CA key
openssl genrsa -aes256 -out root-ca.key 4096

# Create root CA certificate
openssl req -new -x509 -days 3650 -key root-ca.key \
  -out root-ca.crt \
  -subj "/C=US/ST=State/L=City/O=Example Corp/CN=Example Root CA"

# Create intermediate CA key
cd /opt/pki/intermediate-ca
openssl genrsa -aes256 -out intermediate.key 4096

# Create intermediate CA CSR
openssl req -new -key intermediate.key -out intermediate.csr \
  -subj "/C=US/ST=State/L=City/O=Example Corp/CN=Example Intermediate CA"

# Sign intermediate CA with root
openssl x509 -req -days 1825 -in intermediate.csr \
  -CA /opt/pki/root-ca/root-ca.crt \
  -CAkey /opt/pki/root-ca/root-ca.key \
  -CAcreateserial -out intermediate.crt \
  -extensions v3_ca

# Create server certificate for Rancher
openssl genrsa -out /opt/pki/certs/rancher.key 4096

openssl req -new -key /opt/pki/certs/rancher.key -out /opt/pki/certs/rancher.csr \
  -subj "/C=US/ST=State/L=City/O=Example Corp/CN=rancher.example.com"

# Create SAN extension file
cat > /opt/pki/certs/rancher-ext.cnf << 'EOCONF'
[req_ext]
subjectAltName = @alt_names

[alt_names]
DNS.1 = rancher.example.com
DNS.2 = *.rancher.example.com
DNS.3 = rke2-api.example.com
IP.1 = 10.0.1.10
EOCONF

# Sign Rancher cert with intermediate CA
openssl x509 -req -days 730 -in /opt/pki/certs/rancher.csr \
  -CA /opt/pki/intermediate-ca/intermediate.crt \
  -CAkey /opt/pki/intermediate-ca/intermediate.key \
  -CAcreateserial -out /opt/pki/certs/rancher.crt \
  -extfile /opt/pki/certs/rancher-ext.cnf \
  -extensions req_ext

# Create cert bundle (server cert + intermediate)
cat /opt/pki/certs/rancher.crt \
    /opt/pki/intermediate-ca/intermediate.crt \
    > /opt/pki/certs/rancher-bundle.crt

# Create Kubernetes TLS secret
kubectl create secret tls rancher-tls \
  --cert=/opt/pki/certs/rancher-bundle.crt \
  --key=/opt/pki/certs/rancher.key \
  --namespace=cattle-system

# Create CA bundle secret (for Rancher to trust the CA)
kubectl create secret generic tls-ca \
  --from-file=cacerts.pem=/opt/pki/root-ca/root-ca.crt \
  --namespace=cattle-system

# Option B: Use cert-manager with Private CA ClusterIssuer
cat > /tmp/private-ca-issuer.yaml << 'EOF'
# First create the CA secret
# kubectl create secret tls my-ca-secret \
#   --cert=path/to/ca.crt --key=path/to/ca.key \
#   --namespace cert-manager

apiVersion: cert-manager.io/v1
kind: ClusterIssuer
metadata:
  name: private-ca-issuer
spec:
  ca:
    secretName: my-ca-secret
EOF

kubectl apply -f /tmp/private-ca-issuer.yaml

17.1 HAPROXY CONFIGURATION (RECOMMENDED FOR ON-PREM)
------------------------------------------------------
# Install HAProxy on LB node(s)
dnf install -y haproxy

cat > /etc/haproxy/haproxy.cfg << 'EOF'
#------------------------------------------------------------------------------
# HAProxy Configuration for RKE2 + Rancher
# Version: 2.x
#------------------------------------------------------------------------------

global
    log         /dev/log local0
    log         /dev/log local1 notice
    chroot      /var/lib/haproxy
    pidfile     /var/run/haproxy.pid
    maxconn     50000
    user        haproxy
    group       haproxy
    daemon
    
    # SSL settings
    ssl-default-bind-ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384
    ssl-default-bind-options ssl-min-ver TLSv1.2 no-tls-tickets
    
    stats socket /run/haproxy/admin.sock mode 660 level admin expose-fd listeners
    stats timeout 30s

defaults
    mode                    tcp
    log                     global
    option                  tcplog
    option                  dontlognull
    option                  redispatch
    retries                 3
    timeout connect         10s
    timeout client          1m
    timeout server          1m

#------------------------------------------------------------------------------
# STATS INTERFACE (restrict access!)
#------------------------------------------------------------------------------
frontend stats
    bind *:8404
    mode http
    stats enable
    stats uri /stats
    stats refresh 30s
    stats auth admin:REPLACE_WITH_STRONG_PASSWORD
    # Restrict to admin subnet
    acl admin_net src 10.0.1.0/24
    http-request deny unless admin_net

#------------------------------------------------------------------------------
# KUBERNETES API (6443)
#------------------------------------------------------------------------------
frontend kube-api
    bind *:6443
    mode tcp
    option tcplog
    default_backend kube-api-backend

backend kube-api-backend
    mode tcp
    option ssl-hello-chk
    balance roundrobin
    
    option tcp-check
    
    server cp-node-01 10.0.2.11:6443 check inter 10s fall 3 rise 2
    server cp-node-02 10.0.2.12:6443 check inter 10s fall 3 rise 2
    server cp-node-03 10.0.2.13:6443 check inter 10s fall 3 rise 2

#------------------------------------------------------------------------------
# RKE2 REGISTRATION (9345)
#------------------------------------------------------------------------------
frontend rke2-registration
    bind *:9345
    mode tcp
    option tcplog
    default_backend rke2-registration-backend

backend rke2-registration-backend
    mode tcp
    balance roundrobin
    
    server cp-node-01 10.0.2.11:9345 check inter 10s fall 3 rise 2
    server cp-node-02 10.0.2.12:9345 check inter 10s fall 3 rise 2
    server cp-node-03 10.0.2.13:9345 check inter 10s fall 3 rise 2

#------------------------------------------------------------------------------
# HTTPS (443) - Rancher UI
#------------------------------------------------------------------------------
frontend https-in
    bind *:443
    mode tcp
    option tcplog
    default_backend https-backend

backend https-backend
    mode tcp
    balance leastconn
    
    option ssl-hello-chk
    option tcp-check
    
    # Route to worker nodes where ingress controller runs
    server wrk-node-01 10.0.3.21:443 check inter 10s fall 3 rise 2
    server wrk-node-02 10.0.3.22:443 check inter 10s fall 3 rise 2
    server wrk-node-03 10.0.3.23:443 check inter 10s fall 3 rise 2

#------------------------------------------------------------------------------
# HTTP (80) - Redirect and ACME challenges
#------------------------------------------------------------------------------
frontend http-in
    bind *:80
    mode tcp
    option tcplog
    default_backend http-backend

backend http-backend
    mode tcp
    balance roundrobin
    
    server wrk-node-01 10.0.3.21:80 check inter 10s fall 3 rise 2
    server wrk-node-02 10.0.3.22:80 check inter 10s fall 3 rise 2
    server wrk-node-03 10.0.3.23:80 check inter 10s fall 3 rise 2
EOF

# Validate HAProxy config
haproxy -c -f /etc/haproxy/haproxy.cfg

# Enable and start HAProxy
systemctl enable --now haproxy

# Setup keepalived for VIP failover (on both LB nodes)
dnf install -y keepalived

cat > /etc/keepalived/keepalived.conf << 'EOF'
vrrp_script check_haproxy {
    script "kill -0 $(cat /var/run/haproxy.pid)"
    interval 2
    weight -20
    fall 2
    rise 2
}

vrrp_instance VI_1 {
    state MASTER          # BACKUP on second LB node
    interface eth0        # Change to actual interface
    virtual_router_id 51
    priority 100          # 90 on backup node
    advert_int 1
    
    authentication {
        auth_type PASS
        auth_pass REPLACE_WITH_KEEPALIVED_SECRET
    }
    
    virtual_ipaddress {
        10.0.1.10/24      # The VIP
    }
    
    track_script {
        check_haproxy
    }
    
    notify_master "/etc/keepalived/notify.sh MASTER"
    notify_backup "/etc/keepalived/notify.sh BACKUP"
    notify_fault  "/etc/keepalived/notify.sh FAULT"
}
EOF

systemctl enable --now keepalived

    upstream ingress-http {
        least_conn;

SECTION 22: DISASTER RECOVERY PROCEDURE
================================================================================

22.1 SCENARIO 1: SINGLE CONTROL PLANE NODE FAILURE
-----------------------------------------------------
# RKE2 etcd (3-node quorum) tolerates 1 node failure automatically.
# No intervention needed unless node is permanently lost.

# To remove failed node from cluster:
kubectl get nodes
kubectl drain <failed-node> --ignore-daemonsets --delete-emptydir-data
kubectl delete node <failed-node>

# On the replacement node, install RKE2 with same config and rejoin
# (Refer to Section 12.5 for join procedure)

22.2 SCENARIO 2: MAJORITY CONTROL PLANE FAILURE (2+ CP NODES DOWN)
---------------------------------------------------------------------
# etcd has lost quorum. Cluster is read-only / non-functional.
# Must restore etcd from snapshot.

# Step 1: Stop RKE2 on ALL remaining nodes
systemctl stop rke2-server
systemctl stop rke2-agent  # On worker nodes

# Step 2: On the surviving CP node, restore etcd snapshot
# Find available snapshots:
ls /var/lib/rancher/rke2/server/db/snapshots/

# Step 3: Restore (this RESETS etcd to snapshot state)
rke2 server \
  --cluster-reset \
  --cluster-reset-restore-path=/var/lib/rancher/rke2/server/db/snapshots/etcd-snapshot-name

# Step 4: Start RKE2 on the restored node
systemctl start rke2-server

# Step 5: Wait for single-node cluster to be ready
kubectl get nodes

# Step 6: Rejoin other CP nodes
# On the other CP nodes, clear old data and rejoin:
rm -rf /var/lib/rancher/rke2/server/db/
systemctl start rke2-server
# (They will sync from the restored leader)

# Step 7: Verify cluster health
kubectl get nodes
/var/lib/rancher/rke2/bin/etcdctl member list -w table

22.3 VELERO RESTORE PROCEDURE
--------------------------------
# Restore all namespaces from backup
velero restore create --from-backup daily-backup-YYYYMMDDHHMMSS

# Restore specific namespace
velero restore create \
  --from-backup daily-backup-YYYYMMDDHHMMSS \
  --include-namespaces my-application

# Check restore status
velero restore get
velero restore describe <restore-name>

# WHY: The etcd snapshot contains cluster state (deployments, services, configs).
# Velero backups contain PVC data and workload manifests.
# A complete DR requires BOTH: etcd snapshot (cluster state) + Velero (PVC data).

23.2 RKE2 UPGRADE PROCEDURE
------------------------------
# IMPORTANT: Upgrade one node at a time. Never upgrade all nodes simultaneously.
# Upgrade order: CP Node 1 -> CP Node 2 -> CP Node 3 -> Workers

# On EACH control plane node (one at a time):
# 1. Drain the node (for workers; for CP nodes consider if safe)
kubectl cordon <node-name>

# 2. Download new RKE2 version
export NEW_VERSION="v1.28.10+rke2r1"
curl -sfL https://get.rke2.io | INSTALL_RKE2_VERSION=$NEW_VERSION sh -

# 3. Restart RKE2
systemctl restart rke2-server

# 4. Wait for node to rejoin and be Ready
kubectl get nodes -w

# 5. Verify etcd health before proceeding
/var/lib/rancher/rke2/bin/etcdctl endpoint health --cluster -w table

# 6. Uncordon if needed
kubectl uncordon <node-name>

# 7. Proceed to next node

#!/bin/bash
# Cluster Health Check Script
# Run from any CP node with kubeconfig access

export KUBECONFIG=/etc/rancher/rke2/rke2.yaml
export PATH=$PATH:/var/lib/rancher/rke2/bin

echo "============================================"
echo "  RKE2 + RANCHER CLUSTER HEALTH CHECK"
echo "  $(date)"
echo "============================================"

# 1. Node Status
echo ""
echo "--- NODE STATUS ---"
kubectl get nodes -o wide

# Check for NotReady nodes
NOT_READY=$(kubectl get nodes | grep -v "Ready" | grep -v "NAME" | wc -l)
if [ "$NOT_READY" -gt 0 ]; then
  echo "WARNING: $NOT_READY node(s) not in Ready state!"
fi

# 2. etcd Health
echo ""
echo "--- ETCD HEALTH ---"
export ETCDCTL_ENDPOINTS='https://127.0.0.1:2379'
export ETCDCTL_CACERT='/var/lib/rancher/rke2/server/tls/etcd/server-ca.crt'
export ETCDCTL_CERT='/var/lib/rancher/rke2/server/tls/etcd/server-client.crt'
export ETCDCTL_KEY='/var/lib/rancher/rke2/server/tls/etcd/server-client.key'
export ETCDCTL_API=3
etcdctl endpoint health --cluster -w table 2>/dev/null || echo "ERROR: Cannot reach etcd"
etcdctl endpoint status --cluster -w table 2>/dev/null

# 3. System Pods
echo ""
echo "--- SYSTEM PODS ---"
kubectl get pods -n kube-system | grep -v "Running\|Completed"

# 4. Rancher Status
echo ""
echo "--- RANCHER STATUS ---"
kubectl get pods -n cattle-system
kubectl get deployment rancher -n cattle-system

# 5. Certificates
echo ""
echo "--- TLS CERTIFICATES ---"
kubectl get certificates -A

# 6. Storage
echo ""
echo "--- STORAGE ---"
kubectl get storageclass
kubectl get pvc -A | grep -v "Bound"

# 7. Resource Usage
echo ""
echo "--- RESOURCE USAGE ---"
kubectl top nodes 2>/dev/null || echo "metrics-server not available"

# 8. Pending/Failed Pods
echo ""
echo "--- PROBLEM PODS ---"
kubectl get pods -A | grep -v "Running\|Completed\|NAME"

echo ""
echo "============================================"
echo "  Health check complete"
echo "============================================"
SCRIPT

chmod +x /usr/local/bin/cluster-health-check.sh

# Run the health check
cluster-health-check.sh

33.1 GITHUB ACTIONS EXAMPLE
------------------------------
# .github/workflows/deploy.yml
cat << 'EOF'
name: Deploy to Kubernetes

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v4
    
    - name: Configure kubectl
      uses: azure/k8s-set-context@v3
      with:
        method: kubeconfig
        kubeconfig: ${{ secrets.KUBECONFIG }}
    
    - name: Build and push image
      run: |
        docker build -t registry.example.com/my-app:${{ github.sha }} .
        docker push registry.example.com/my-app:${{ github.sha }}
    
    - name: Deploy
      run: |
        kubectl set image deployment/my-app \
          my-app=registry.example.com/my-app:${{ github.sha }} \
          -n my-app
        kubectl rollout status deployment/my-app -n my-app --timeout=300s
    
    - name: Verify deployment
      run: |
        kubectl get pods -n my-app
        kubectl get ingress -n my-app
EOF

35.2 ANSIBLE PLAYBOOK - NODE PREPARATION
------------------------------------------
# playbooks/prepare-nodes.yaml
cat << 'EOF'
---
- name: Prepare Oracle Linux 9 nodes for RKE2
  hosts: all
  become: yes
  vars:
    rke2_version: "v1.28.8+rke2r1"
  
  tasks:
  - name: Update all packages
    dnf:
      name: "*"
      state: latest
      update_cache: yes
  
  - name: Install required packages
    dnf:
      name:
        - bash-completion
        - bind-utils
        - chrony
        - conntrack
        - curl
        - ethtool
        - ipset
        - iscsi-initiator-utils
        - jq
        - lvm2
        - nfs-utils
        - openssl
        - policycoreutils-python-utils
        - socat
        - tar
        - vim
        - wget
      state: present
  
  - name: Disable swap
    command: swapoff -a
    when: ansible_swaptotal_mb > 0
  
  - name: Remove swap from fstab
    replace:
      path: /etc/fstab
      regexp: '^(\s*)([^#\n]+\s+)(\w+\s+)swap(\s+.*)$'
      replace: '#\1\2\3swap\4'
  
  - name: Load required kernel modules
    modprobe:
      name: "{{ item }}"
      state: present
    loop:
      - br_netfilter
      - overlay
      - ip_tables
      - nf_conntrack
  
  - name: Persist kernel modules
    copy:
      dest: /etc/modules-load.d/rke2.conf
      content: |
        br_netfilter
        overlay
        ip_tables
        nf_conntrack
  
  - name: Configure sysctl for Kubernetes
    sysctl:
      name: "{{ item.key }}"
      value: "{{ item.value }}"
      sysctl_set: yes
      state: present
      reload: yes
    loop:
      - { key: "net.ipv4.ip_forward", value: "1" }
      - { key: "net.bridge.bridge-nf-call-iptables", value: "1" }
      - { key: "net.bridge.bridge-nf-call-ip6tables", value: "1" }
      - { key: "vm.swappiness", value: "0" }
      - { key: "fs.inotify.max_user_watches", value: "1048576" }
      - { key: "net.netfilter.nf_conntrack_max", value: "1048576" }
  
  - name: Install container-selinux
    dnf:
      name: container-selinux
      state: present
  
  - name: Configure chrony NTP
    template:
      src: chrony.conf.j2
      dest: /etc/chrony.conf
    notify: restart chronyd
  
  - name: Enable and start services
    systemd:
      name: "{{ item }}"
      state: started
      enabled: yes
    loop:
      - chronyd
      - iscsid
      - auditd
  
  - name: Configure firewalld for worker nodes
    firewalld:
      port: "{{ item }}"
      permanent: yes
      state: enabled
    loop:
      - "10250/tcp"
      - "8472/udp"
      - "30000-32767/tcp"
    when: inventory_hostname in groups['workers']
  
  - name: Configure firewalld for control plane nodes
    firewalld:
      port: "{{ item }}"
      permanent: yes
      state: enabled
    loop:
      - "6443/tcp"
      - "9345/tcp"
      - "2379-2380/tcp"
      - "10250/tcp"
      - "8472/udp"
    when: inventory_hostname in groups['control_plane']
  
  - name: Reload firewalld
    command: firewall-cmd --reload
  
  handlers:
  - name: restart chronyd
    systemd:
      name: chronyd
      state: restarted
EOF

39.1 PROMETHEUS ALERTING RULES
---------------------------------
cat > /tmp/rke2-alerts.yaml << 'EOF'
apiVersion: monitoring.coreos.com/v1
kind: PrometheusRule
metadata:
  name: rke2-cluster-alerts
  namespace: cattle-monitoring-system
spec:
  groups:
  - name: cluster.health
    rules:
    - alert: NodeNotReady
      expr: kube_node_status_condition{condition="Ready",status="true"} == 0
      for: 5m
      labels:
        severity: critical
      annotations:
        summary: "Node {{ $labels.node }} is not ready"
        description: "Node has been not ready for more than 5 minutes"

    - alert: EtcdHighDiskUsage
      expr: etcd_mvcc_db_total_size_in_bytes / 1024 / 1024 / 1024 > 6
      for: 10m
      labels:
        severity: warning
      annotations:
        summary: "etcd database size exceeds 6GB on {{ $labels.instance }}"
        description: "etcd DB is {{ $value | humanize }}GB. Compact etcd before it hits 8GB limit."

    - alert: EtcdLeaderChange
      expr: changes(etcd_server_leader_changes_seen_total[15m]) > 1
      labels:
        severity: warning
      annotations:
        summary: "etcd leader changed more than once in 15 minutes"

    - alert: APIServerHighLatency
      expr: histogram_quantile(0.99, sum(rate(apiserver_request_duration_seconds_bucket[5m])) by (verb, le)) > 1
      for: 5m
      labels:
        severity: warning
      annotations:
        summary: "Kubernetes API server 99th percentile latency > 1s"

    - alert: PVCAlmostFull
      expr: (kubelet_volume_stats_available_bytes / kubelet_volume_stats_capacity_bytes) * 100 < 15
      for: 5m
      labels:
        severity: warning
      annotations:
        summary: "PVC {{ $labels.persistentvolumeclaim }} in namespace {{ $labels.namespace }} is {{ $value | humanize }}% full"

    - alert: CertificateExpiringSoon
      expr: certmanager_certificate_expiration_timestamp_seconds - time() < 604800
      labels:
        severity: warning
      annotations:
        summary: "Certificate {{ $labels.name }} in namespace {{ $labels.namespace }} expires in less than 7 days"

    - alert: RancherDeploymentUnavailable
      expr: kube_deployment_status_replicas_available{namespace="cattle-system",deployment="rancher"} < 2
      for: 5m
      labels:
        severity: critical
      annotations:
        summary: "Rancher deployment has fewer than 2 replicas available"

    - alert: NodeHighMemoryUsage
      expr: (1 - (node_memory_MemAvailable_bytes / node_memory_MemTotal_bytes)) * 100 > 85
      for: 10m
      labels:
        severity: warning
      annotations:
        summary: "Node {{ $labels.instance }} memory usage is {{ $value | humanize }}%"

    - alert: NodeHighCPUUsage
      expr: 100 - (avg by(instance) (rate(node_cpu_seconds_total{mode="idle"}[5m])) * 100) > 85
      for: 10m
      labels:
        severity: warning
      annotations:
        summary: "Node {{ $labels.instance }} CPU usage is {{ $value | humanize }}%"
EOF

40.1 ETCD PERFORMANCE TUNING
------------------------------
# In /etc/rancher/rke2/config.yaml, etcd-arg settings:
# etcd-arg:
#   - "quota-backend-bytes=8589934592"      # 8 GB max DB size
#   - "auto-compaction-mode=periodic"        # Compact every N hours
#   - "auto-compaction-retention=8h"         # Keep 8 hours of revisions
#   - "snapshot-count=5000"                  # Snapshot frequency
#   - "heartbeat-interval=100"               # ms - increase for slow networks
#   - "election-timeout=1000"                # ms - 10x heartbeat

# Manual etcd compaction (if DB grows too large):
export ETCDCTL_API=3
CURRENT_REVISION=$(/var/lib/rancher/rke2/bin/etcdctl endpoint status \
  -w json | jq '.[0].Status.header.revision')
/var/lib/rancher/rke2/bin/etcdctl compact $CURRENT_REVISION
/var/lib/rancher/rke2/bin/etcdctl defrag

40.2 KERNEL PERFORMANCE TUNING
---------------------------------
# CPU scaling governor (performance mode)
for CPU in /sys/devices/system/cpu/cpu*/cpufreq/scaling_governor; do
  echo "performance" > $CPU 2>/dev/null || true
done

# Make persistent via tuned
dnf install -y tuned
tuned-adm profile throughput-performance
systemctl enable --now tuned

# Disable transparent hugepages (required for some workloads)
echo never > /sys/kernel/mm/transparent_hugepage/enabled
echo never > /sys/kernel/mm/transparent_hugepage/defrag

# Persist via rc.local or systemd service:
cat > /etc/systemd/system/disable-thp.service << 'EOF'
[Unit]
Description=Disable Transparent Huge Pages
After=local-fs.target

[Service]
Type=oneshot
ExecStart=/bin/sh -c 'echo never > /sys/kernel/mm/transparent_hugepage/enabled'
ExecStart=/bin/sh -c 'echo never > /sys/kernel/mm/transparent_hugepage/defrag'

[Install]
WantedBy=multi-user.target
EOF
systemctl enable --now disable-thp

================================================================================
IMPORTANT NOTES AND COMMON PITFALLS
================================================================================

PITFALL 1: etcd on slow storage
  SYMPTOM: "etcdserver: request timed out" errors, leader elections
  SOLUTION: Use NVMe SSD. Run fio benchmark BEFORE installation.

PITFALL 2: Clock drift between nodes
  SYMPTOM: Certificate validation errors, etcd peer rejection
  SOLUTION: Verify chrony sync on ALL nodes. Alert at > 100ms drift.

PITFALL 3: Firewalld blocking pod-to-pod communication
  SYMPTOM: Services unreachable within cluster
  SOLUTION: Ensure masquerading is enabled. Check that VXLAN (8472/UDP) is allowed.

PITFALL 4: SELinux denying container operations
  SYMPTOM: Pods fail to start with "permission denied"
  SOLUTION: Install rke2-selinux package BEFORE installing RKE2.

PITFALL 5: Insuffficient inotify limits
  SYMPTOM: "too many open files", monitoring agents crash
  SOLUTION: Set fs.inotify.max_user_watches=1048576 in sysctl.

PITFALL 6: etcd DB exceeds 8GB quota
  SYMPTOM: API writes rejected with "etcdserver: mvcc: database space exceeded"
  SOLUTION: Enable auto-compaction. Monitor DB size. Manual compact if needed.

PITFALL 7: Joining nodes with wrong token
  SYMPTOM: "token is already expired" or "failed to find peer"
  SOLUTION: Token must be identical on ALL nodes. Use same config.yaml.

PITFALL 8: Let's Encrypt rate limits
  SYMPTOM: "too many certificates" error from ACME
  SOLUTION: Use staging issuer for testing. Move to prod issuer only for final setup.

PITFALL 9: NodePort conflicts with existing services
  SYMPTOM: Service fails to start, port already in use
  SOLUTION: Check for conflicting services on ports 30000-32767 before installation.

PITFALL 10: Missing br_netfilter module at boot
  SYMPTOM: NetworkPolicy not working, pods can't communicate across nodes
  SOLUTION: Add to /etc/modules-load.d/rke2.conf for persistence across reboots.
